bool  autoReport,
bool  exitAfter,
bool  loop

static

Belongs to the group of functions to measure performance in the source code. Indicates the starting point of the measurement. A description and example can be found below the detailed description of this function.

Parameters
numberOfMeasurementsThe number of times performanceMeasurementStart() and performanceMeasurementFinish() have to be reached until the report is created. Default is 1.
ignoreFirstNumberYou can give a number of loops, not taken into account in the calculation. This is usefull, if there are some routines that initialise the first time they run. Default is 0.
autoReportIf autoReport is set TRUE, you don't need the function performanceMeasurementReport(). In this case the report is generated as soon as the given number of measurements is reached. Default is false.
exitAfterIf exitAfter is set TRUE, the exit(0) command is called after output of the report. Default is false.
loopIf loop is set TRUE, the measurement will start again after output of the report. From the second measurement the parameter ignoreFirstNumber is ignored. Default is false.
Returns
Returns TRUE if the actual call is measured, FALSE if not.

Instruction for use:

To measure the performance of a source code part, you have to use at least three functions if you use default settings. performanceMeasurementStart() is the startingpoint of the measurement. At the end of the measured code you have to insert the function performanceMeasurementFinish(). At a place of your choice in your code place the function performanceMeasurementReport() to get the output data.

Here is an example using default settings to measure the time a function consumed. 

int main()
{
    // start the time measurement;
    performanceMeasurementStart();

    functionThatDoesBrilliantThings();

    // end the measurement
    performanceMeasurementFinish();

    // output report
    performanceMeasurementReport();
}

This is an example of the report for the above code. 

Performance data:
Ignored first:          0
Number of measurements: 1
Total time:   4.740 ms
Minimal time: 4.740 ms
Maximum time: 4.740 ms
Average time: 4.740 ms

Let's explain the output with changed settings. In this example glfwSwapBuffers(window) runs only every 1/60 second. To measure the time for the rest of the routines in the loop, the performanceMeasurementStart() function is called after, the performanceMeasurementFinish() function before the command with the delay. 

while(!vu.getWindowShouldClose()){
    [...]
    functionThatDoesBrilliantThings();
    [...]
    JdPerformance::performanceMeasurementFinish();
    glfwSwapBuffers(window); // runs every 1/60 sec (erwin_hp with geforce on)
    JdPerformance::performanceMeasurementStart(150, 5, true, false, true);
}

Compares the running time of two algorithms, each in a loop for one second.

Instructions for use:

To compare to versions of an algorithm attach them to two functions. JdPerfomanceTest will compare them. Use the following templates:

Declare the functions. 

void jdPerfTestAlgo1();
void jdPerfTestAlgo2();

Define the functions. 

void jdPerfTestAlgo1(){
    // *****************************************************************************
    // declare the variables your first algorithm (1) uses here

    // your declaration ends here
    // *****************************************************************************

    // *****************************************************************************
    // put in the first algorithm (1) here

    // your algorithm (1) ends here
    // *****************************************************************************
}

void jdPerfTestAlgo2(){
    // *****************************************************************************
    // declare the variables your second algorithm (2) uses here

    // your declaration ends here
    // *****************************************************************************

    // *****************************************************************************
    // put in the second algorithm (2) here

    // your algorithm (2) ends here
    // *****************************************************************************
}

In your main function declare the function pointers. 

void (*p_jdPerfTestAlgo1)()=jdPerfTestAlgo1;
void (*p_jdPerfTestAlgo2)()=jdPerfTestAlgo2;

Now you can call the test and compare. 

JdPerformance::performanceTest(p_jdPerfTestAlgo1, p_jdPerfTestAlgo2);

Here is an example I used to optimize a simulation function. In this case I found a faster method to calculate the cosinus between to vectors. 

#include <iostream>
#include "glm/glm.hpp"
#include "glm/gtc/matrix_transform.hpp"
#include "glm/gtc/type_ptr.hpp"

void jdPerfTestAlgo1();
void jdPerfTestAlgo2();

void jdPerfTestAlgo1(){
    // *****************************************************************************
    // declare the variables your first algorithm (1) uses here

    glm::vec3 v1, v2;
    v1 = glm::vec3(2.3f, 5.1f, 0.2f);
    v2 = glm::vec3(6.3f, 2.1f, 3.4f);

    // your declaration ends here
    // *****************************************************************************

    // *****************************************************************************
    // put in the first algorithm (1) here

    glm::dot(glm::normalize(v1), glm::normalize(v2));

    // your algorithm (1) ends here
    // *****************************************************************************
}

void jdPerfTestAlgo2(){
    // *****************************************************************************
    // declare the variables your second algorithm (2) uses here

    glm::vec3 v1, v2;
    v1 = glm::vec3(2.3f, 5.1f, 0.2f);
    v2 = glm::vec3(6.3f, 2.1f, 3.4f);

    // your declaration ends here
    // *****************************************************************************

    // *****************************************************************************
    // put in the second algorithm (2) here

    glm::dot(v1, v2)/(glm::length(v1) * glm::length(v2));

    // your algorithm (2) ends here
    // *****************************************************************************
}
